Cloud computing can mean several different things. Like most definitions in this world, different things are returened depending on who you ask.

"The Cloud" is the usual place to start. And to build a website, you usually need to start with "The Cloud".

To build a website, you will need a dedicated server that will be fast, reliable, and secure. Unless your skill set is going to be systems administrator and/or network programmer, you probably do not want to worry about the 

		$ sudo apachectl start
calls. You will not want to worry about the letsEncrpy Secure Socket Layer (SSL) Certificates. You do not want to fandangle with email accounts and 16 years of boilerplate code to create another GUI to another bash processed sql bridge.

By choosing someone to host your server for you, hours can be saved on your project. Having a server and space in "The Cloud" is like calling get_server('my-website.com'); in the game of life. But before you go, there are three final concepts that we need to touch on.

  1. Infrastructure as a service (IaaS)
  2. Platform as a service (PaaS)
  3. Software as a service (SaaS)

IaaS are servers meant to handle high level API calls online. Consider a company like PayPal where a shop can make an API call asking for money from one account to be put into their account. Consider My Fitness Pal which can look up recipes where get_calories_by_serving(recipe) < _GET['calories'] . MFP and PayPal (along with practically every top tech company) has an amazing API structure. And some companies ONLY have API's and databases. The website is not their focus. Creating is not their focus. These companies are 100% focused on databases.

PaaS are servers meant to provide a platform where users can come and create content. WYSIWIGS, Google App Engine, Oracle Cloud Platform are all examples of PaaS. And there are many, many more. Within this group, we have five other specific example:

  1. Public Platforms
  2. Private Platforms
  3. Hybrid Platforms
  4. Communications Platforms
  5. Mobile Platforms
  6. Open Platforms

SaaS . This is the holy grail. These companies offer their clients a service. Say the client has 10 gigs of data. Company Foo just made software bar.exe. And this new software will take that 10 gig data import, process it, and spit out a website, which Company Foo has also taken care of for hosting, security, and everything else. In this case, the client rarely sees the code that executes and does all of the work. Things just magically happen, and everyone appears to be left happy at the end of the day.
